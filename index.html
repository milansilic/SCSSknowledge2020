SASS Syntactically Awesome Style Sheets

...is a CSS preprocessor that significantly improves syntax clarity and functionality, enabling fewer lines of code. Newer syntax, SCSS Sassy CSS uses CSS block formatting: { } braces to denote code blocks and semicolons to separate rules within a block.

VARIABLES are named-memory-locations for storing values, often those we want to use many times. By placing a value inside the variable, we set the same value anywhere we used that particular variable! 
$myColor1: green;

We create a Variable by declaring a name ("$" symbol + custom name) and a value that will be transferred anywhere we "call" that variable. 

p {
   color: $myColor1;
}

We use a Variable by writing full variable name instead of value within the declaration. In this example we "passed" the value "green" to the property "color".

NESTING is a possibility to place a set of rules of an element, directly inside a set of rules of their parent element. With this we achieve HTML visual hierarchy, which provides better orientation, visualisation and cleaner code. 

article {
   background: grey;
   p {
      width: 400px;
   }
}

Nesting an Element "p" inside element "article". We refered to all "p" elements that are descendants of any "article" element in the HTML code. "Width" value from the example won't apply to "p" elements outside the "article" elements. 

a {
   color: grey;
   &:hover {
      color: white;
   }
}

Nesting a Pseudo Class using symbol "&" which replace the name of the selector we are in. In this example "&:hover" is actually compiled to "a:hover". 

main {
  width: 100%;
  @media screen (min-width:768px)
    width: 60%;
  }
}

Nesting a Media Query directly into the block of code of an element we want to affect. This way we have a better view of how one element changes, depending on media queries.

EXTEND is a SASS directive that allows us to use all defined declarations of one selector, into another. We get fewer lines of code because there is no need to repeat declarations that make up one code block, when we need to use them in another. 

.button {
   padding: 15px 30px;
   text-align: center;
   font-size: 16px;
   cursor: pointer;
}

For example, the set of declarations we want to use multiple times iz located in the class "button". 

.submit {
   @extend .button;
   background-color: green;
   color: white;
}

Inside class "submit" code block, we import all declarations from class "button" using expression "@extend", followed by the name of class "button". This way, class "submit" will contain all declarations from class "button" + its own.

MIXIN is a SASS directive for making reuseable code blocks. Inside a mixin we can define particular style once, which can be used multiple times. To use or "call" created mixin, we use "@include" directive. 

@mixin myMixin1 {
   width: 400px;
   background: green;
}

We create a Mixin by "@mixin" directive and a custom mixin name. Inside curly braces we define all the declarations that will constitute a mixin. 

.box {
   @include myMixin1;
   font-size: 16px;
}

We call a Mixin by "@include" directive, followed by a mixin name. In this example we imported declarations "width" and "background" from the mixin "myMixin1", into the class "box".

MIXINS() with arguments alow us to modify them every time we call them. They are extremely convenient when we want to use same sets of atributes with different values, inside different code blocks. 
@mixin myRadius1($vred) {
  -webkit-border-radius: $vred;
  -moz-border-radius: $vred;
  -o-border-radius: $vred;
  border-radius: $vred;
}

We create a Mixin() with arguments as usual, and enter an argument inside parentheses, in the form of variable. We use arguments from the parentheses, instead of mixin property values that we want to set later, when we call a mixin. 

.box {
   @include myRadius1(3px);
}

We call a Mixin() with arguments as usual, and set a value inside ()prentesies, which will be transfered to the argument inside mixin ()prentesies, and into mixin atributes, where we used that argument. 

@mixin Fle($juCo, $flWr) {
  display: flex;
  justify-content: $juCo;
  flex-wrap: $flWr;
}

We create a Mixin() with multiple arguments by declaring them in parentheses, separated by commas. Now we have the ability to manipulate various values for multiple attributes, every time we call a mixin! 

.container {
  @include Fle(center, wrap);
}

We call a mixin() with multiple parameters by seting the values in parentheses, which will be transferred in the same order to the arguments in mixin parentheses, and into mixin atributes, where we used that arguments.
