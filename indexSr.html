...je CSS preprocesor koji značajno poboljšava preglednost i funkcionalnost sintakse, omogućujući kraći i jasniji kod. Novija sintaksa, SCSS Sassy CSS koristi CSS formatiranje blokova: { } zagrade za blokove koda i tačka-zareze za odvajanje pravila unutar bloka.

VARIJABLE su imenovane memorijske lokacije za čuvanje vrednosti koje želimo da upotrebimo više puta. Određivanjem vrednosti u varijabli, određujemo vrednost na svakom mestu gde smo varijablu upotrebili: 
$mojaBoja1: green;

Pravimo Varijablu simbolom "$", ispred proizvoljnog imena. Zatim unosimo vrednost koju će ta varijabla preneti svuda gde bude upotrebljena! 

p {
   color: $mojaBoja1;
}

Koristimo Varijablu navodeći puno ime varijable umesto same vrednosti za željeni atribut! U ovom primeru smo atributu "color" preneli vrednost "green".

NESTING je "ugnježđavanje" elemenata koje stilizujemo, direktno u blokove koda "roditelj" elemenata. Tako postižemo vizuelnu hijerarhiju kao u HTML-u, što nam olakšava orijentaciju, preglednost i pojednostavljuje kod: 

article {
   background: grey;
   p {
      width: 400px;
   }
}

"Ugnjezdili" smo Element "p" u element "article". Na taj način smo stilizovali samo elemente "p" koji se u HTML-u nalaze u elementima "article". Vrednost atributa "width" iz primera, neće važiti za "p" elemente koji se ne nalaze u elementu "article"! 

a {
   color: grey;
   &:hover {
      color: white;
   }
}

"Ugnjezdili" smo Pseudo Klasu koristeći simbol "&" koji zamenjuje naziv selektora u čijem bloku koda se nalazimo. U ovom primeru "&:hover" zapravo znači "a:hover". 

main {
  width: 100%;
  @media screen (min-width:768px)
    width: 60%;
  }
}

"Ugnjezdili" smo Media Query direktno u blok koda elementa na kog delujemo. Tako imamo jasniju sliku o svim promenama na jednom elementu, u zavisnosti od media query-ja.

EKSTEND je SASS direktiva koja omogućuje korišćenje svih definisanih deklaracija jednog selektora, u drugom. Dobijamo manje linija koda jer ne moramo ponovo unositi deklaracije koje čine jedan blok koda, kada su nam potrebne u drugom. 

.button {
   padding: 15px 30px;
   text-align: center;
   font-size: 16px;
   cursor: pointer;
}

Recimo da klasa "button" sadrži deklaracije koje želimo da budu deo nekog drugog bloka koda. 

.submit {
   @extend .button;
   background-color: green;
   color: white;
}

U bloku koda klase "submit", izrazom "@extend", praćenim imenom klase "button", uvozimo sve deklaracije iz klase "button"! Na ovaj način, klasa "submit" će sadržati sve deklaracije klase "button" + svoje lične.

MIKSIN je SASS direktiva za kreiranje ponovo upotrebljivog bloka koda. Unutar miksina definišemo određeni stil, kog možemo upotrebljavati više puta. Direktivom "@include" pozivamo kreirani miksin gde želimo. 

@mixin mojMiksin {
   width: 400px;
   background: green;
}

Pravimo miksin izrazom "@mixin" i proizvoljnim imenom miksina. Nakon toga u vitičastim zagradama određujemo sve deklaracije koje će činiti miksin. 

.box {
   @include mojMiksin;
   font-size: 16px;
}

Pozivamo miksin izrazom "@include" nakon kog navodimo ime miksina. U ovom primeru smo u klasu "box" ubacili deklaracije "width" i "background", iz miksina "mojMiksin".

MIKSINI() sa argumentima dozvoljavaju da ih modifikujemo svaki put kada ih pozovemo. Izuzetno su korisni kada želimo da upotrebljavamo iste setove atributa sa različitim vrednostima, u različitim blokovima koda. 
@mixin mojRadius($vred) {
  -webkit-border-radius: $vred;
  -moz-border-radius: $vred;
  -o-border-radius: $vred;
  border-radius: $vred;
}

Pravimo Miksin() sa argumentima standardno, unoseći argument (prenosioc vrednosti) u obliku varijable, unutar ()zagrada. Koristimo argument iz zagrade umesto vrednosti atributa u mixinu, koju ćemo odrediti tek kad miksin budemo pozivali. 

.box {
   @include mojRadius(3px);
}

Pozivamo Miksin() sa argumentima standardno, zadajući željenu vrednost u ()zagrade, koja se prenosi u argument iz ()zagrade miksina, a samim tim i u vrednosti atributa miskina, gde smo taj argument upotrebili. 

@mixin Fle($juCo, $flWr) {
  display: flex;
  justify-content: $juCo;
  flex-wrap: $flWr;
}

Pravimo mixin() sa više argumenata navodeći ih u zagradama, odvojeno zarezima. Sada imamo mogućnost da manipulišemo različitim vrednostima za veći broj atributa, svaki put kada pozivamo miksin. 

.container {
  @include Fle(center, wrap);
}

Pozivamo miksin() sa više argumenata: određujući vrednosti u zagradama, koje će se istim redosledom prenositi u argumente u zagradi miksina, a samim tim i u vrednosti odgovarajućih atributa miksina.
